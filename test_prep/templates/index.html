1. Identity and Access Management (IAM)

  • Users: Mapped to physical users, has password for AWS Console access
  • Groups: Contains users only
  • Policies: JSON documents outlining permissions
  • Roles: For EC2 instances or AWS services
  • Security: MFA + Password Policy
  • AWS CLI: Manage AWS services using command-line
  • AWS SDK: Manage AWS services using programming languages
  • Access Keys: Access AWS using CLI or SDK
  • Audit: IAM Credential Reports & IAM Access Advisor

2. Elastic Compute Cloud (EC2)

  • EC2 Instance: AMI (OS) + Instance Size (CPU + RAM) + Storage + Security Groups + EC2 User Data
  • Security Groups: Firewall attached to EC2 instances
  • EC2 User Data: Script launched at the first start of an instance
  • SSH: Start a terminal into EC2 Instances (port 22)
  • EC2 Instance Role: Link to IAM roles
  • Purchasing Options: On-Demand, Spot, Reserved, Dedicated Host, Dedicated Instance
  • Storage: EBS, EFS, FSx, Instance Store
  • AMI: Pre-configured EC2 instances for faster setup

3. High Availability & Scalability

  • Elastic Load Balancing (ELB): Distribute traffic across backend EC2 instances
  • Auto Scaling Groups (ASG): Implement Elasticity for applications across multiple AZ
  • Vertical Scaling: Increase instance size
  • Horizontal Scaling: Increase number of instances
  • High Availability: Run instances across multiple AZ
  • ELB Types: Application LB (HTTP â€“ L7), Network LB (TCP â€“ L4), Classic LB (old)

4. Amazon S3 (Simple Storage Service)

  • Buckets vs Objects: Global unique name, tied to a region
  • S3 Security: IAM policy, S3 Bucket Policy (public access), S3 Encryption
  • S3 Websites: Host static websites
  • Versioning: Multiple versions for files, prevent accidental deletes
  • Access Logs: Log requests within S3 bucket
  • Replication: Same-region or cross-region, requires versioning
  • Storage Classes: Standard, IA, 1Z-IA, Intelligent, Glacier, Glacier Deep Archive
  • Lifecycle Rules: Transition objects between classes
  • Glacier Vault Lock / S3 Object Lock: WORM (Write Once Read Many)
  • Snow Family: Import data onto S3 through physical devices, edge computing
  • Storage Gateway: Hybrid solution to extend on-premises storage to S3

5. Databases & Analytics

  • Relational Databases (OLTP): RDS & Aurora (SQL)
  • In-memory Database: ElastiCache
  • Key/Value Database: DynamoDB (serverless) & DAX (cache for DynamoDB)
  • Data Warehouse (OLAP): Redshift (SQL)
  • Hadoop Cluster: EMR
  • Athena: Query data on Amazon S3 (serverless & SQL)
  • QuickSight: Dashboards on your data (serverless)
  • DocumentDB: "Aurora for MongoDB" (JSON â€“ NoSQL database)
  • QLDB: Financial Transactions Ledger (immutable journal, cryptographically verifiable)
  • Managed Blockchain: Managed Hyperledger Fabric & Ethereum blockchains
  • Glue: Managed ETL and Data Catalog service
  • Database Migration: DMS
  • Neptune: Graph database

6. Serverless

  • Lambda: Function as a Service, seamless scaling, reactive
  • API Gateway: Expose Lambda functions as HTTP API
  • Docker: Container technology to run applications
  • ECS: Run Docker containers on EC2 instances
  • Fargate: Run Docker containers without provisioning infrastructure (serverless)
  • ECR: Private Docker Images Repository
  • Batch: Run batch jobs on AWS across managed EC2 instances
  • Lightsail: Predictable & low pricing for simple application & DB stacks

7. Deployment & Developer Tools

  • CloudFormation: Infrastructure as Code for AWS resources
  • Beanstalk: Platform as a Service (PaaS) for certain languages or Docker
  • CodeDeploy: Deploy & upgrade applications on servers
  • Systems Manager: Patch, configure, run commands at scale
  • OpsWorks: Managed Chef and Puppet in AWS
  • CodeCommit: Private git repository
  • CodeBuild: Build & test code
  • CodePipeline: Orchestrate from code to build to deploy
  • CodeArtifact: Store software packages / dependencies
  • CodeStar: Unified view for CICD and code
  • Cloud9: Cloud IDE with collaboration
  • AWS CDK: Define cloud infrastructure using programming languages

8. Global Applications

  • Route 53: Global DNS service
  • CloudFront: Content Delivery Network (CDN)
  • S3 Transfer Acceleration: Fast, global file transfers
  • AWS Global Accelerator: Improve global application performance
  • Outposts: Extend AWS to on-premises data centers
  • WaveLength: Edge computing for 5G networks
  • Local Zones: Bring AWS resources closer to users

9. Integration & Messaging

  • SQS: Queue service with multiple producers/consumers
  • SNS: Notification service with multiple subscribers
  • Kinesis: Real-time data streaming, persistence, and analysis
  • Amazon MQ: Managed Apache ActiveMQ

10. Cloud Monitoring

  • CloudWatch: Metrics, alarms, logs, and events
  • CloudTrail: Audit API calls within AWS account
  • CloudTrail Insights: Automated analysis of CloudTrail Events
  • X-Ray: Trace requests through distributed applications
  • Service Health Dashboard: AWS service status across regions
  • Personal Health Dashboard: Personalized AWS health events
  • Amazon CodeGuru: Automated code reviews and application performance recommendations

11. VPC & Networking

  • VPC: Virtual Private Cloud
  • Subnets: Tied to AZ, network partition of VPC
  • Internet Gateway: Provide internet access at VPC level
  • NAT Gateway / Instances: Internet access for private subnets
  • NACL: Stateless subnet rules for inbound/outbound
  • Security Groups: Stateful, operate at EC2 instance or ENI level
  • VPC Peering: Connect VPCs with non-overlapping IP ranges
  • VPC Endpoints: Private access to AWS Services within VPC
  • VPC Flow Logs: Network traffic logs
  • Site to Site VPN: Connect on-premises to AWS
  • Direct Connect: Private connection to AWS
  • Transit Gateway: Connect thousands of VPCs and on-premises networks

12. Security & Compliance

  • Shared Responsibility Model
  • Shield: DDoS Protection
  • WAF: Web Application Firewall
  • KMS & CloudHSM: Encryption key management
  • Certificate Manager: SSL/TLS Certificates
  • Artifact: Access to compliance reports
  • GuardDuty: Malicious behavior detection

  • Config: Resource configuration tracking
  • Macie: Sensitive data discovery in S3
  • Security Hub: Centralized security management
  • Detective: Root cause analysis for security issues
  • AWS Abuse: Report abusive or illegal resource usage
  • Root user privileges: Account management, billing changes

13. Machine Learning

  • Transcribe: Speech to text

  • Translate: Language translation

  • Connect: Cloud contact center
  • Comprehend: Natural language processing
  • SageMaker: End-to-end machine learning platform
  • Forecast: Time-series forecasting
  • Kendra: ML-powered search engine
  • Personalize: Real-time recommendations
  • Textract: Extract text and data from documents

14. Account Management & Billing

  • Organizations: Multi-account management
  • Control Tower: Set up and govern multi-account environments
  • Systems Manager: Operational insights across accounts
  • Trusted Advisor: Best practice recommendations
  • Cost management tools: Compute Optimizer, Pricing Calculator, Cost Explorer, Budgets
  • Savings Plans: Flexible pricing model for compute savings
  • Cost Anomaly Detection: Identify unusual spending patterns
  • Service Quotas: Manage AWS service limits

15. Identity Federation

  • IAM: Identity and Access Management within AWS
  • Organizations: Manage multiple AWS accounts
  • Security Token Service (STS): Temporary, limited-privilege credentials
  • Cognito: User management for web/mobile apps
  • Directory Services: Integrate with Microsoft Active Directory
  • IAM Identity Center: Single sign-on for multiple AWS accounts and applications

16. Well-Architected Framework

  • Security: Protect information and systems
